# Fetchr

**Browser-native download manager** — a fusion of IDM and JDownloader, built for your homelab.

```
browser extension  ──────────────────────►  local agent (FastAPI)
  intercepts downloads                         yt-dlp + aiohttp
  media scanner / batch select                 queue + WebSocket
  dark popup UI                                saves to ~/Downloads/Fetchr
```

---

## Structure

```
fetchr/
├── agent/              ← Python backend (runs on your PC)
│   ├── main.py         ← FastAPI server  (port 9876)
│   ├── downloader.py   ← HTTP + yt-dlp engine
│   ├── queue_manager.py← Queue, pause/resume, state
│   ├── requirements.txt
│   ├── install.sh      ← Linux/macOS installer + systemd unit
│   └── install.bat     ← Windows installer + Task Scheduler
│
└── extension/          ← Chrome extension (Manifest V3)
    ├── manifest.json
    ├── background.js   ← Service worker, download intercept
    ├── content.js      ← Media scanner, batch selector overlay
    ├── popup/          ← Main UI (popup.html + popup.js + popup.css)
    ├── options/        ← Settings page
    └── icons/          ← Auto-generated by install script
```

## Quick Start

### 1 — Install the agent

**Linux / macOS**
```bash
cd fetchr/agent
chmod +x install.sh
./install.sh
```

**Windows**
```
Double-click: fetchr\agent\install.bat
```

Both scripts:
- Create a Python virtualenv and install dependencies
- Generate the extension icons
- Register an auto-start service (systemd on Linux, Task Scheduler on Windows)
- Start the agent immediately on `http://127.0.0.1:9876`

### 2 — Load the extension in Chrome

1. Open `chrome://extensions`
2. Enable **Developer mode** (top-right toggle)
3. Click **Load unpacked**
4. Select the `fetchr/extension/` folder
5. The Fetchr icon appears in your toolbar

### 3 — You're ready

- Every browser download is automatically redirected to the agent
- Click the toolbar icon to see the queue
- Right-click any link → **Download with Fetchr**
- Click **Grab media** in the popup to scan the current page for video/audio
- Click **Batch select** to pick multiple links/images from the page

## Agent API

| Method | Path | Description |
|--------|------|-------------|
| GET | `/health` | Agent status |
| POST | `/downloads` | Add download |
| GET | `/downloads` | List all |
| POST | `/downloads/{id}/pause` | Pause |
| POST | `/downloads/{id}/resume` | Resume |
| DELETE | `/downloads/{id}` | Cancel + remove |
| POST | `/scan` | Scan URL for media formats (yt-dlp) |
| GET/PUT | `/settings` | Read/update settings |
| WS | `/ws` | Real-time progress stream |

### Add a download manually
```bash
curl -X POST http://127.0.0.1:9876/downloads \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com/file.zip"}'
```

### Add a media download (yt-dlp)
```bash
curl -X POST http://127.0.0.1:9876/downloads \
  -H "Content-Type: application/json" \
  -d '{"url":"https://www.youtube.com/watch?v=...", "dl_type":"media"}'
```

## Configuration

Edit via the extension's **Settings** page (gear icon), or directly:

```json
// agent/settings.json (auto-created on first save)
{
  "save_path":      "C:/Users/Ugo/Downloads/Fetchr",
  "max_concurrent": 3,
  "max_speed_kbps": 0,
  "intercept_all":  true
}
```

## Requirements

- **Python 3.10+**
- **Chrome / Chromium** (Manifest V3)
- `yt-dlp` (installed automatically) — needed for media/video downloads
- Internet access from the agent machine

## Proxmox note

If you later want to move the agent to a Proxmox LXC:
1. Change `AGENT_BASE` in `background.js` and `popup.js` from `127.0.0.1:9876` to your LXC IP
2. Expose port 9876 (or use a reverse proxy / Nginx)
3. Add the LXC IP to `host_permissions` in `manifest.json`
